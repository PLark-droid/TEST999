# miyabi FAQ - よくある質問

困ったときはここを確認してください。

---

## 基本的な質問

### Q: miyabiって何？

**A:** AI駆動開発フレームワークです。

```
従来: 人間がコードを書く
miyabi: 人間は「何を作りたいか」を書く → AIがコードを書く
```

### Q: 本当にコードを書かなくていいの？

**A:** はい。Issueを書くだけです。

```
1. Issueを書く（日本語でOK）
2. 「処理して」と言う
3. PRが来る
4. マージする

コードは1行も書きません。
```

### Q: プログラミングの知識は必要？

**A:** 最低限でOK。

**必須:**
- Markdown（Issueを書くため）
- Git基礎（マージの概念）

**不要:**
- プログラミング言語の詳細
- フレームワークの知識
- テストの書き方

## トラブルシューティング

### Q: Agentが動かない

**A:** 以下を確認：

```bash
# 1. 環境変数
echo $GITHUB_TOKEN
# → 値が表示されればOK

# 2. リポジトリアクセス
gh repo view
# → リポジトリ情報が表示されればOK

# 3. miyabiステータス
miyabi status
# → 状態を確認
```

### Q: PRが来ない

**A:** 以下を確認：

1. Issueは作成されていますか？
   ```bash
   gh issue list
   ```

2. 処理を開始しましたか？
   ```
   「Issue #X を処理して」
   ```

3. エスカレーション待ちではないですか？
   - Agentから質問が来ていないか確認

### Q: PRの品質が低い

**A:** Issueの書き方を見直しましょう。

**悪い例:**
```
ログイン機能を作って
```

**良い例:**
```
## 概要
メールアドレスとパスワードでログインする機能

## やりたいこと
- POST /auth/login エンドポイント
- JWTトークンを返す
- パスワードはbcryptでハッシュ化

## 制約
- セッションは24時間で切れる
```

### Q: エラーメッセージが出た

**A:** エラーメッセージをそのままClaude Codeに伝えてください。

```
「このエラーが出た: {エラーメッセージ}」
```

一緒に解決します。

## エスカレーションについて

### Q: エスカレーションって何？

**A:** Agentが人間に判断を委ねることです。

```
Agent: 「認証方式はJWTとセッションどちらにしますか？」
人間: 「JWTで」
Agent: 「了解。続行します」
```

これは正常な動作です。

### Q: エスカレーションにどう答えればいい？

**A:** 分かる範囲で答えてください。

- 分かる → 判断を伝える
- 分からない → 「分からない」と言う
- 任せたい → 「おすすめで」と言う

### Q: エスカレーションが多すぎる

**A:** Issueをより具体的に書きましょう。

制約や条件を明記すると、Agentが自分で判断できます。

## コマンド関連

### Q: よく使うコマンドは？

**A:**

```bash
# 毎日使う
miyabi status          # 状態確認
gh issue create        # Issue作成
gh pr list             # PR一覧
gh pr merge <番号>      # マージ

# 学習用
/miyabi-learn          # チュートリアル
/miyabi-issue-help     # Issue作成サポート
/miyabi-help           # このFAQ
```

### Q: miyabi autoって何？

**A:** 全自動モードです。

```bash
miyabi auto
```

未処理のIssueを自動で検出して処理し続けます。
放置しておけばPRが来ます。

## 品質について

### Q: 品質スコアって何？

**A:** ReviewAgentがつける点数です。

```
100点満点
80点以上で合格
```

**減点項目:**
- TypeScriptエラー: -30点
- ESLintエラー: -20点
- 脆弱性: -40点

### Q: 品質スコアが80点未満だった

**A:** Agentが自動で修正します。

人間は待つだけでOK。
3回失敗したらエスカレーションが来ます。
