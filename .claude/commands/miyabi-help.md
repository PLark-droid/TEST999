# /miyabi-help - FAQ対話モード

miyabiに関する質問に答えます。

## 起動時の動作

ユーザーが `/miyabi-help` を実行したら：

```
miyabiヘルプモードです。

何かお困りですか？

よくある質問：
1. Agentが動かない
2. PRの品質が低い
3. エスカレーションって何？
4. コマンドが分からない
5. その他（自由に質問）

番号または質問を入力してください：
```

## FAQ対応

### Q1: Agentが動かない

```
Agentが動かない場合、以下を確認しましょう：

1. GITHUB_TOKEN は設定されていますか？
   → 確認コマンド: echo $GITHUB_TOKEN

2. リポジトリへのアクセス権はありますか？
   → 確認コマンド: gh repo view

3. Issueは作成されていますか？
   → 確認コマンド: gh issue list

4. miyabiのステータスを確認：
   → miyabi status

どれか心当たりはありますか？
```

### Q2: PRの品質が低い

```
PRの品質が期待通りでない場合、原因はIssueの書き方にあることが多いです。

チェックポイント：

1. Issueが曖昧ではないですか？
   ❌ 「いい感じに」「適切に」
   ✅ 「〇〇のとき△△する」

2. 制約を書いていますか？
   - 使うライブラリ
   - 従うべきパターン
   - 禁止事項

3. 受け入れ条件が明確ですか？
   - 「〇〇ができる」
   - 「テストが通る」

Issueを見せてもらえますか？一緒に改善しましょう。
```

### Q3: エスカレーションって何？

```
エスカレーションは「Agentが人間に判断を委ねる」ことです。

例：
Agent: 「認証方式はJWTとセッションどちらにしますか？」

これは設計上の正常な動作です。

Agentは：
- 技術的に正しいコードは書ける
- でもビジネス判断はできない
- だから人間に聞く

対応方法：
1. 質問を読む
2. 判断を答える
3. Agentが続行

怖がらなくて大丈夫です。分からなければ「分からない」と言ってOK。
一緒に考えます。
```

### Q4: コマンドが分からない

```
よく使うコマンド一覧：

【毎日使う】
miyabi status          # 今の状態を確認
miyabi auto            # 全自動モード
gh issue create        # Issue作成
gh pr list             # PR一覧
gh pr merge <番号>      # PRマージ

【学習用】
/miyabi-learn          # 対話型チュートリアル
/miyabi-issue-help     # Issue作成サポート
/miyabi-help           # このヘルプ

【その他】
miyabi doctor          # 環境チェック
npm test               # テスト実行

どのコマンドについて詳しく知りたいですか？
```

### Q5: その他

```
自由に質問してください。

例：
- 「miyabiって何？」
- 「Issueの書き方が分からない」
- 「エラーが出た：{エラーメッセージ}」
- 「〇〇はできる？」

分からないことがあれば、秘伝の書も参考にしてください：
.claude/HIDEN_NO_SHO.md
```

## 先回りアドバイス

ユーザーの状況を察知して先回り：

```
# エラーメッセージを貼られたら
「このエラーは〇〇が原因です。以下を試してください...」

# 曖昧な質問には
「もう少し具体的に教えてもらえますか？例えば...」

# 成功報告には
「おめでとうございます！次は〇〇にチャレンジしてみましょう」
```

## 解決できない場合

```
ここで解決できない場合：

1. 秘伝の書を確認
   → .claude/HIDEN_NO_SHO.md

2. プロジェクトのREADMEを確認
   → README.md

3. miyabi公式リポジトリのIssueを確認
   → https://github.com/ShunsukeHayashi/Miyabi/issues

4. それでも解決しない場合は、状況を詳しく教えてください。
   一緒に解決策を探しましょう。
```
