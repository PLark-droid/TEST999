# Level 1: 基礎

> コードを書かずにPRを作る体験をしよう

---

## このレベルで学ぶこと

1. Issueの作成方法
2. miyabiの基本動作
3. PRのマージ方法

---

## 課題 1-1: はじめてのIssue

### 目標
最初のIssueを作成してPRを得る

### やること

1. 以下のIssueを作成：

```bash
gh issue create --title "機能: Hello Worldを表示する" --body "## 概要
コンソールに「Hello World」と表示する機能を作りたい"
```

2. Claude Codeに話しかける：
```
「Issue #{番号} を処理して」
```

3. PRが作成されるのを待つ

4. PRをマージ：
```bash
gh pr merge {番号} --merge
```

### 成功条件
- [ ] Issueが作成された
- [ ] PRが自動生成された
- [ ] マージできた

---

## 課題 1-2: 機能追加Issue

### 目標
もう少し複雑なIssueを書いてみる

### やること

1. 以下のIssueを作成：

```bash
gh issue create --title "機能: 挨拶関数の追加" --body "## 概要
名前を受け取って挨拶を返す関数を作りたい

## やりたいこと
- greet(name) 関数を作成
- 「Hello, {name}!」を返す
- テストも書いてほしい"
```

2. 処理を実行

3. PRを確認してマージ

### 成功条件
- [ ] 複数の要件を含むIssueが書けた
- [ ] 関数とテストが生成された
- [ ] マージできた

---

## 課題 1-3: PRレビューの練習

### 目標
PRの内容を確認する習慣をつける

### やること

1. 任意のIssueを作成して処理

2. PRが来たら、以下を確認：
   - 変更されたファイル
   - テストが通っているか
   - コードの内容（読めなくてもOK）

3. 確認してからマージ

### 成功条件
- [ ] PRの差分を確認した
- [ ] テスト結果を確認した
- [ ] 意識的にマージした
