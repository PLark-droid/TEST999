## 課題 3-1: 並列Issue処理

### 目標
複数のIssueを同時に処理する

### やること

1. 3つのIssueを連続作成：

```bash
# Issue 1
gh issue create --title "機能: 日付フォーマット関数" --body "## 概要
日付を様々な形式でフォーマットする関数"

# Issue 2
gh issue create --title "機能: 数値フォーマット関数" --body "## 概要
数値を通貨形式やパーセント形式でフォーマットする関数"

# Issue 3
gh issue create --title "機能: 文字列フォーマット関数" --body "## 概要
文字列のパディングやトリミングを行う関数"
```

2. 並列処理を実行：

```
「Issue #X, #Y, #Z を並列で処理して」
```

または：

```bash
miyabi auto
```

3. 複数のPRが同時に作成されるのを確認

### 成功条件
- [ ] 3つのIssueを作成した
- [ ] 並列処理を実行した
- [ ] 複数PRが同時に来た
- [ ] 全てマージした

### 学び
```
並列処理 = 時間効率の最大化
待ち時間に別のIssueが処理される
```

## 課題 3-2: チーム展開計画

### 目標
チームメンバーにmiyabiを教える準備をする

### やること

1. 新人向け説明資料のIssueを作成：

```bash
gh issue create --title "ドキュメント: チーム向けmiyabi導入ガイド" --body "## 概要
新しくチームに参加したメンバー向けのmiyabi導入ガイドを作成

## 含める内容
- miyabiとは何か（1段落で）
- 最初にやること（3ステップ）
- よくある質問（5つ）
- 困ったときの連絡先"
```

2. 処理してPRを確認

3. 内容をレビュー：
   - 分かりやすいか？
   - 足りない情報はないか？

4. 必要なら修正指示を出す

### 成功条件
- [ ] 説明資料が生成された
- [ ] 内容をレビューした
- [ ] チームに共有できる状態になった

---

## 課題 3-3: 自由プロジェクト

### 目標
自分でアプリを設計し、miyabiで実装する

### やること

1. 作りたいものを考える（例）：
   - シンプルなTODOアプリ
   - メモ管理ツール
   - 計算機
   - 何でもOK

2. 必要な機能をIssueに分解：
   - Issue 1: 基本機能
   - Issue 2: 追加機能
   - Issue 3: テスト・ドキュメント

3. 順番に処理（または並列処理）

4. 全てマージして完成

### 成功条件
- [ ] 自分でアプリを企画した
- [ ] Issueに分解した
- [ ] miyabiで実装した
- [ ] 動くものが完成した
