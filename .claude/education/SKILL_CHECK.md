# miyabi習熟度チェックリスト

自分の習熟度を確認しましょう。

---

## Level 1: 基礎スキル

### Issue作成
- [ ] `gh issue create` コマンドが使える
- [ ] タイトルと本文を書ける
- [ ] 「概要」「やりたいこと」の形式で書ける

### miyabi操作
- [ ] `miyabi status` で状態を確認できる
- [ ] Claude Codeに「処理して」と指示できる
- [ ] PRが来るまで待てる

### PR操作
- [ ] PRの一覧を確認できる (`gh pr list`)
- [ ] PRの差分を見られる
- [ ] PRをマージできる (`gh pr merge`)

### 心構え
- [ ] コードを書かなくていいことを理解した
- [ ] 待つことの重要性を理解した

**Level 1 達成: 上記の80%以上にチェック**

---

## Level 2: 実践スキル

### Issue作成（応用）
- [ ] 複数の要件を1つのIssueにまとめられる
- [ ] 制約条件を明記できる
- [ ] テスト要件を書ける
- [ ] 曖昧な表現を避けられる

### エスカレーション
- [ ] エスカレーションが来ても慌てない
- [ ] 質問を理解して判断を伝えられる
- [ ] 分からない時は「分からない」と言える

### 品質確認
- [ ] PRの品質スコアを確認できる
- [ ] TypeScript/ESLintエラーの有無を確認できる
- [ ] テストカバレッジを確認できる

**Level 2 達成: 上記の80%以上にチェック**

---

## Level 3: 応用スキル

### 効率化
- [ ] 複数Issueを並列処理できる
- [ ] `miyabi auto` を活用できる
- [ ] 待ち時間を有効活用できる

### 設計
- [ ] 大きな機能をIssueに分解できる
- [ ] 依存関係を意識できる（書かなくてもOK）
- [ ] 優先順位をつけられる

### チーム展開
- [ ] miyabiを他の人に説明できる
- [ ] 新人の質問に答えられる
- [ ] トラブルシューティングできる

**Level 3 達成: 上記の80%以上にチェック**

---

## マスターレベル

### 全てに該当する
- [ ] miyabiなしの開発に戻りたくない
- [ ] 「何を作るか」に集中できている
- [ ] コードを読む時間より考える時間が長い
- [ ] チームの生産性が上がった

### 次にやるべきこと

**Level 1未達成の場合:**
→ `/miyabi-learn` で基礎から始める

**Level 2未達成の場合:**
→ エスカレーション体験を積む
→ より具体的なIssueを書く練習

**Level 3未達成の場合:**
→ 並列処理を試す
→ チームメンバーに教えてみる

**マスター未達成の場合:**
→ 日常業務でmiyabiを使い続ける
→ 時間が解決する
